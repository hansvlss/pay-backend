第 0 步：免责声明与准备

风险提醒：刷机有风险。请先备份原始固件和 boot 分区。下列步骤均需在你确认可以承担风险时执行。

第 1 步：准备（Win/Mac/Linux）

  1. 下载工具：WinSCP、PuTTY（或在 Mac 用 ssh）
  2. 为安全起见在本地创建一个工作目录：mkdir ~/ax6000-backup
  3. 准备一个可以访问路由器的网线与静态 IP（说明见下面）

第 2 步：获取 stok / 进入开发者模式（命令 & 示例）

示例命令（把 {token} 换成你实际获得的 stok）:

http://192.168.31.1/cgi-bin/luci/;stok={token}/api/misystem/set_sys_time?timezone=...

（视频中方法请参照主文档）

第 3 步：设置 SSH 并保持生存（脚本）

把下面脚本保存为 auto_ssh.sh 并上传到 /tmp，然后运行：

echo -e 'admin\nadmin' | passwd root
nvram set ssh_en=1
nvram set telnet_en=1
nvram set uart_en=1
nvram set boot_wait=on
nvram commit
sed -i 's/channel=.*/channel="debug"/g' /etc/init.d/dropbear
/etc/init.d/dropbear restart

第 4 步：刷入 U-Boot（在线方式）

cd /tmp && curl -s -O https://fastly.jsdelivr.net/gh/miaoermua/unlock-redmi-ax6000@main/setup.sh && chmod +x setup.sh && ./setup.sh

如需离线：上传 uboot.sh 到 /tmp 并运行。

第 5 步：备份与恢复要点

  • 刷入前请先备份分区：示例命令 dd if=/dev/mtdblockX of=/tmp/backup-mtdX.bin bs=4M
  • 备份文件通过 WinSCP 下载到本地保存

常见故障排查

1. 设备无法启动 → 尝试从备份恢复 U-Boot、或使用串口进入恢复模式。
2. 无法登录 SSH → 使用串口或 WinSCP 检查 /etc/passwd 与 dropbear.
