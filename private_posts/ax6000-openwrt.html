第 0 步：免责声明

风险提醒：刷机有风险，可能导致设备不可用。请先备份原始固件与 boot 分区，并确认你能自行承担风险。

第 1 步：准备（Windows / macOS / Linux）

  1. 下载工具：WinSCP、PuTTY（或 macOS 的 ssh）
  2. 在本地创建工作目录：mkdir ~/ax6000-backup
  3. 准备一根网线并设置静态 IP（示例见下）

第 2 步：进入开发者模式并获取 stok

示例（把 {token} 替换成你实际的 stok）：

http://192.168.31.1/cgi-bin/luci/;stok={token}/api/misystem/set_sys_time?timezone=...

第 3 步：开启 SSH 并保持（脚本）

保存为 auto_ssh.sh 并上传到 /tmp，然后运行：

echo -e 'admin\nadmin' | passwd root
nvram set ssh_en=1
nvram set telnet_en=1
nvram set uart_en=1
nvram set boot_wait=on
nvram commit
sed -i 's/channel=.*/channel="debug"/g' /etc/init.d/dropbear
/etc/init.d/dropbear restart

第 4 步：刷入 U-Boot（在线方式）

cd /tmp && curl -s -O https://fastly.jsdelivr.net/gh/miaoermua/unlock-redmi-ax6000@main/setup.sh && chmod +x setup.sh && ./setup.sh

第 5 步：备份与恢复

  • 备份分区示例：dd if=/dev/mtdblockX of=/tmp/backup-mtdX.bin bs=4M
  • 通过 WinSCP 下载备份到本地保存

常见故障排查

1. 设备无法启动 → 尝试从备份恢复 U-Boot。
2. 无法登录 SSH → 使用串口或 WinSCP 检查 /etc/passwd 与 dropbear。
